# Unity asmdef: Referencing Assembly-CSharp from Test Assemblies

## Problem

In Unity 6 (6000.x), putting `"Assembly-CSharp"` in an `.asmdef` file's `references` array does **NOT** create a working reference to the default predefined assembly. This affects test assemblies that need to reference game scripts.

## Symptoms

- CS0246 errors: "The type or namespace name 'X' could not be found"
- Only affects types from `Assembly-CSharp`; types from other asmdef-defined assemblies resolve fine
- The `.csproj` generated by Unity will NOT contain a `<ProjectReference>` to `Assembly-CSharp.csproj`
- `Assembly-CSharp.dll` compiles successfully and contains all expected types

## Root Cause

Unity's asmdef `references` array is designed for **asmdef-to-asmdef** references. `Assembly-CSharp` is a **predefined** assembly (not defined by any `.asmdef`), so Unity's compiler and project generator ignore it.

## Verified In

- Unity 6000.4.2f1

## Solution

Create an `.asmdef` for your game scripts so the test asmdef can reference it as a proper asmdef-to-asmdef dependency:

### 1. Create `Assets/Scripts/GameScripts.asmdef`

```json
{
    "name": "GameScripts",
    "rootNamespace": "",
    "references": [],
    "includePlatforms": [],
    "excludePlatforms": [],
    "allowUnsafeCode": false,
    "overrideReferences": false,
    "precompiledReferences": [],
    "autoReferenced": true,
    "defineConstraints": [],
    "versionDefines": [],
    "noEngineReferences": false
}
```

### 2. Update test asmdef to reference it

```json
{
    "name": "EditModeTests",
    "references": [
        "UnityEngine.TestRunner",
        "UnityEditor.TestRunner",
        "GameScripts"
    ],
    ...
}
```

### Caveats

- Creating a game scripts asmdef changes the compilation unit for ALL game scripts. Any scripts that reference types in other asmdefs (or Editor-only APIs) may need their own asmdef or `includePlatforms` adjustments.
- If game scripts reference types from packages that are `autoReferenced: false`, you'll need to add those package asmdef names to the game scripts asmdef's `references` array.
- Third-party folders in Assets that are NOT wrapped by the new asmdef will remain in `Assembly-CSharp` and won't be accessible from `GameScripts` unless also referenced.

## Alternative: Keep Tests That Don't Need Game Types

If creating a game scripts asmdef is too disruptive, limit test files to only reference:
- Types defined in other asmdef-controlled assemblies
- Unity built-in types
- Types with no dependencies on Assembly-CSharp
